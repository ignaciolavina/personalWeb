LH APP [Failed]

about a failed story, and learnigns outcomes.

Some years ago I realized about the necessity of a software solution for managing the sports center where I was working. At that time the company was starting and didn't have any kind of software for managing the bookings, the shoop, and other business issues, meaning some inefficiencies and unnecessary time spent on repetitive tasks that could be automatized.

That year I started my new studies, a double degree in informatics engineering and business administration, and started to learn my first Object Oriented programming language.

The union of that two facts gave, as a result, the decision of "learn by doing", making a new personal project, the LH APP, a web application for managing the sports center. 


FIRST ATTEMPT
Sep 2015 – Jun 2016

On this first attempt, I started the project with the idea of learning the Java programming language. The motivation behind was that I had a programming course at the university about Object Programming language, which was driven in Java.

Since it was my first programming language, I was a newbie and I haven,t any kind of programming background, so it was a real challenge to try this with no knowledge at all. Also at that time, I didn't know about the existence of any kind of version control like Git or Subversion, so when I finalized, the code was a mess. I have lots of code copies in folders with many version names (yes, exactly what you are thinking). When I discovered Git was incredible, I still wonder how many problems and hours of work I would be saved.

Hey! but not everything was negative! The experience for me was awesome, and I had many great outcomes: A lot of Java Knowledge, an Honor mention on the Programming course, and lots of code, documentation and great ideas.

Here are the main learnings about that project:

Failure causes:

  • Wrong language code chosen (Java).
  • The absence of knowledge about version management (Git, Subversion or others).
  • The absence of project tracking and project flow.
  • Unorganized documentation process.
  • Inexperience.

Main outcomes:

  • More than 12 thousand lines of code.
  • A lot of documentation and Ideas.
  • Experience and knowledge.

An Honor mention award in the programming course.

SECOND ATTEMPT
sept 2017 - February 2018

After the first failure and a couple of years with the project stored on my desk, I decided to retake the project. This time I decided to join a group of five university students, and instead of Java do it in web programming, (make sense, isn't it?).

Again the project resulted in a new failure experience, but once more with a lot of new learnings as outcome and positive experience:

Failure causes:

  • Lack of time. Most of the team members were on the last year of studies, with a lot of courses (as it is a double degree) and some of them started to work. This had, as a result, the incapacity of a continuous workflow
  • Scheduling problems. The group was diverse, two components had class on the morning session for the university classes, other two had afternoon session and the last one was studying on USA. This made difficult to have team meetings and work together.

Outcomes

  • Learnings and experience on web programming.
  • A lot of documentation and code.
  • Great team experience.

Why writing this post?

The reason of this post is simple, to remind me (and you if you still reading this) that it doesn't matter if you fail because you will learn a lot in the process. Also try to know always why did you fail, and what are the main outcomes of that.

And of course, encourage you to have always a personal project on dev, whatever the topic it is.

At the moment I,m writing this article the project has been stored again in with many other personal projects and ideas for develope, but I'm sure one day It will be retaken.
